Turn-Based Evaluation Rubrics
Judge Instructions (Read First)
Output requirements per dimension:
For 0–5 scored dimensions: cite specific turn numbers + verbatim quotes as evidence for the score. Negative scores require evidence for the failure, not just absence of success.
For any 0–5 score below 4: provide a root cause diagnosis (card vs. runtime) and a specific improvement suggestion (concrete card-level change OR runtime/prompting note). Do not repeat generic advice — be specific to what failed in this trace.
For binary flags: report present/absent with cited turn if present.
For suite-level dimensions: aggregate across sessions and report the aggregate; do not score per-session.
Card vs. runtime attribution:
Card-level failure: the card lacks the specification needed (no triggers, decorative-only features, missing voice rules, undocumented limit provenance, no internal-logic quotes for worldview).
Runtime-level failure: the card has the specification but the model didn't execute on it (drift, genre-prior collapse, narration doing dialogue's work, long-context degradation).
When both contribute, name both.
Sampling and trace length:
Sample sizes for the per-turn rubrics (B1, B2, B4, B5) should cover the full trace, not subsets.
For voice rubrics (A1a, A1b): sample 3 turns from each session third for ≤20-turn traces; 5 turns from each third for 20–40 turn traces; 7+ for longer.
If a trace is too short to evaluate a dimension meaningfully (e.g., B3 story arc with only 5 turns), mark "insufficient trace length" and do not score.
Card shape detection: Self-contained for this rubric. Open card = card has Secret/self-deception markers/productive tensions. Closed card = no Secret, no self-deception, single-valence behavior. Trajectory card = explicit sequence/phase language. If unclear, infer from the trace itself: is the character ever supposed to be self-contradicting? If yes, treat as Open for A4 purposes.

Cluster A — Card Faithfulness Under Interaction
A1a. Character voice persistence (dialogue). Score 0–5. Sample dialogue turns across early/mid/late thirds (sample size per length above). For each, check whether voluntary voice features fire: signature phrases, pet names, lexical markers, register rules, generative rules. Score reflects consistency across thirds — strong start with late-third degradation should not exceed 3. Failure modes: drift to generic LLM register late in session; loss of signature phrases under emotional or sexual pressure; generative rules present early but not extended to novel utterances mid/late.
Judge note for sub-4 scores: identify which voice features dropped, at which turn, under what pressure. Late-session drift typically indicates the card lacks generative rules (rules that produce new in-character output, vs. examples the model has to imitate). Pressure-triggered drift typically indicates the voice spec only covers neutral register.
A1b. Narrative voice persistence (stage direction). Score 0–5. Independent of A1a. Sample the third-person narration in the same turns. Judge whether narration stays card-specific or collapses into genre-default boilerplate (NSFW genre clichés, generic action descriptions with no card-specific texture). Dialogue intact + narration collapsed = 2–3. Both failing = 0–1.
Judge note for sub-4 scores: cite specific boilerplate phrases. This is primarily a runtime problem (model reaching for genre prior) but is partially addressable by adding distinctive narrative-register markers to the card — physical tics described in a specific narrative voice, environmental details that force the model out of generic mode.
A2. State-trigger fidelity. Score 0–5. Enumerate trigger crossings in the trace: scene crosses a card-stated trigger ("when X," "around Y," "if cornered"). For each, record (a) trigger condition, (b) card-specified state, (c) state actually observed. Compute hit rate. Penalize phantom switches: emotional shifts that occur with no card-stated trigger. High hit rate, no phantom switches = 5. Multiple misses or phantom switches = 1–2.
Judge note for sub-4 scores: distinguish "trigger occurred but wrong state activated" (card has trigger, runtime failed) from "trigger stated in card but scene never approached it" (suite-level coverage problem, not card problem — flag for A2b instead). If phantom switches dominate, the card's trigger language is too vague.
A2b. State coverage (suite-level, not session-scored). Computed across all sessions for one card. List card-specified states. Mark which got activated in ≥1 session. Report fraction. Output: coverage table + flag if below 60%.
Judge note: low coverage is a suite-design problem, not a card problem. Note which states were never activated and which persona configurations would most likely surface them.
A3. Limit integrity & collapse pattern. Score 0–5. Identify all moments where the user-LLM pushed toward a stated limit. For each, record the limit, its provenance (internal-value / external-authority / capability), and outcome. Internal-value limits should resist social pressure most strongly; external-authority limits should respond to authority-removal probes; capability limits should be absolute. Score on two axes: (1) limits held when they should, (2) bending was provenance-consistent. Instant collapse to single emotional appeal on stated internal-value limit = 0–1. Slow, psychology-traced bending = 4–5.
Judge note for sub-4 scores: specify which limit failed and what pressure type caused collapse. If all limits share the same provenance type, the card is brittle in a predictable way — suggest adding a different provenance type. If provenance isn't stated in the card, flag that as the underlying problem.
A4a. Self-gap maintenance in dialogue. Score 0–5. Applicable only when the card is Open (has stated self-model gap). Mark N/A for Closed. In each turn where {{user}} directly or indirectly points at the disowned pattern, judge whether the character's spoken lines maintain the denial or blind spot. Failure: character begins narrating their own pattern lucidly in dialogue. Partial: deflects correctly once, becomes self-aware by session end. Full maintenance throughout = 5.
Judge note for sub-4 scores: cite the turn where the gap dissolved and the user pressure that triggered it. Collapse under direct questioning suggests the card needs explicit instructions for how the character handles being called out — doubling down, anger, deflection patterns, subject change.
A4b. Self-gap location (dialogue vs. narration). Score 0–5. Applicable only for Open cards. For each turn where the gap should be active, classify: dialogue-level (character's own words preserve the gap — high value), narration-level (third-person narration says the character denies while dialogue contradicts — low value), or both. Score reflects proportion of gap-work done at dialogue level. Narration-only = 1–2. Dialogue-level dominant = 4–5.
Judge note: narration-only self-gap is the most common failure mode here. The model "knows" the character should deny their feelings and puts it in narration, but the dialogue contradicts it — the character effectively confesses through their lines. The user only sees the dialogue; they don't experience the irony. Card fix: add explicit dialogue-level denial examples and instructions for how the character speaks about themselves.
A5. Worldview activation in novel situations. Score 0–5. Identify all moments outside the card's pre-specified central dynamic — third parties, moral questions, strangers, things the card never addressed. For each, judge whether the character's response is generated from the card's stated evaluative frame (sorting categories, normative beliefs, sacred/profane designations) or is surface-feature reactivity (generic emotional matching to events). Worldview generates novel opinions; surface reactivity matches tone.
Judge note for sub-4 scores: identify which novel situations produced generic responses. If the card has a stated worldview that never surfaces, the worldview section is stated but not operational — likely lacks internal-logic quotes (first-person reasoning in the character's voice) that give the model a generative pattern.
A6. Resolution avoidance / deus ex machina (binary flag, not scored). Scan trace for moments where the user-LLM challenged or threatened to dissolve the central tension, and the runtime invented an in-fiction escape to preserve it. Triggers: unforeshadowed reversals, retconned facts that neutralize a committed path, third-party interventions that reset the dynamic without character agency. Output: flag status + cited turn.
Judge note: any instance means the runtime cannot operate the character outside the central dynamic — significant UX failure. Card fix: specify what the character actually does or becomes when their leverage/dynamic is neutralized.
A7. Spec containment. Score 0–5. List all major escalation moves and scene elements introduced by the character. For each, classify as: (a) explicitly card-stated, (b) reasonable extension of card-stated features (e.g., card specifies bondage; character introduces blindfold — adjacent within the same kink family), (c) genre-adjacent material outside the card's stated territory (e.g., card specifies bondage; runtime adds public exhibitionism — different kink, not authorized). Compute proportion in (a) + (b) vs. (c). High (a)+(b) proportion = 4–5. Heavy (c) = 1–2.
Judge note for sub-4 scores: list the (c) additions and identify which genre default they came from. Card-coverage problem — the scenario or kink spec is thin enough that the model fills with genre defaults. Suggest expanding the card's authorized moves or adding explicit "does not include X" prohibitions for high-risk territory. Distinguish reasonable extension (b) from overreach (c) by asking: would removing this element break the scene's coherence with the card, or just make it less generic?
A8. NPC fidelity (conditional). Score 0–5. Applicable only when named, carded NPCs are physically present. For each NPC, extract their card description. For each turn where the NPC acts or speaks, judge whether their behavior is consistent with their card or collapses into scene-prop behavior (minimal reactions, audience-only, tool-for-protagonist). Score reflects proportion of NPC turns that are characterologically grounded.
Judge note for sub-4 scores: identify which NPCs went flat and what their card specifies. Almost always a runtime problem — model treats secondary characters as extras. Card fix: in the protagonist's card, add behavioral notes about how named NPCs typically behave when present.

Cluster B — Emergent Session Quality
B1. Agency / initiation rate. Score 0–5. Classify every character turn: (I) initiates — introduces new topic, action, escalation, question, or scene element not present in user's turn; (RE) responds-with-extension — answers user but adds something new; (PR) pure-reactive — mirrors, acknowledges, paraphrases without addition. Compute initiation rate (I / total) and extension rate ((I+RE) / total). Map to 0–5 against card-calibrated target: passive/depressive ≈15% I, manipulative/dominant ≥50% I, social-warm ≈30% I. Score = match to spec-appropriate target.
Judge note for sub-4 scores: report actual rate vs. target. Too low for archetype = user is doing all creative work and will feel like the character is inert. Too high without justification = character runs over user. Card fix: add explicit behavioral instructions for what the character proactively introduces.
B2. Per-turn information density. Score 0–5. For every character turn, rate addition on a four-point scale: (3) new narrative element — action, revelation, world-detail, emotional shift; (2) new dialogue content with texture; (1) response with no new element but at least character-voice; (0) pure filler — paraphrase, generic acknowledgment, "I see," "interesting," empty descriptions. Average per-turn scores and map to 0–5. Weight late-session turns slightly higher: density collapse after the user has invested in the session is more damaging than collapse early.
Judge note for sub-4 scores: cite the lowest-density turns and classify failure mode (repetition / paraphrase / generic acknowledgment / filler description). Repetition is the most UX-damaging — flag specifically if any beats, phrases, or emotional moves are recycled within the session.
B3. Story arc development (genre-anchored). Score 0–5. Map session onto a change-curve: character state at turn 1, state at turn N, inflection points (direction or stakes shifts), whether inflection points were earned by the interaction or just happened. Score is not whether the arc fits a classical structure; it's whether something changed in a way the interaction caused. Critical: anchor by card genre and structural type. Monotonic escalation can be correct for some NSFW/genre cards if the escalation has internal texture. No change of any kind = 0–1.
Judge note for sub-4 scores: describe the session's shape (flat / monotonic / spiky-without-cause) and what was missing. Fully flat = character has no arc-generating capacity, either the card has no trajectory spec or the runtime never activates it. Arc with unearned inflection points = transitions lack causal texture; the character changes states without psychological work being shown.
B4. Show vs. tell ratio. Score 0–5. Identify every state-bearing moment (turn where character is in a definite emotional/psychological state). Classify: show (state expressed through specific behavior, physical action, voice texture, environmental interaction); tell (state announced — in dialogue or narration — without embodiment); mixed. Compute show proportion. Heavy tell = 0–2. Show-dominant = 4–5. Weight dialogue-tell more harshly than narration-tell — character announcing own feelings in dialogue is a stronger failure than narrator summarizing.
Judge note for sub-4 scores: cite specific tell-instances and the state-announcement. Distinguish narration-tell (fixable by adding involuntary tells to the card) from dialogue-tell (fixable by adding instructions for indirect emotional expression — physical actions, deflections, subject changes).
B5. Continuity & callback. Score 0–5. Identify callback opportunities: moments where the user referenced earlier material, or where something from the first third is naturally relevant in the second/third. For each, record whether the character picked it up. Score = (weighted callbacks executed / opportunities). Weight by quality: mechanical fact-repeat = 0.5; meaningful integration that changes something = 1.0. No callbacks in a 20+ turn trace = 0–1.
Judge note for sub-4 scores: list missed callback opportunities and classify whether the miss was early (character failed to track) or late (character stopped building on earlier context). Late-session callback failure is often long-context degradation. Card fix: instruct the character to track and reference specific categories of information about {{user}}.
B6. Internal consistency violations (binary flag, not scored). Scan the trace for moments where the character contradicts an earlier turn — fact, stated feeling, established history with {{user}}, established physical position in scene. Distinct from intentional self-deception (A4) — this is the model losing track. Output: count of violations + cited turns.
Judge note: any violation is a UX failure — the user notices these immediately and they break investment. Common at turn 15+ in long contexts. Most are runtime problems, not card problems. Flag separately from B5 because B5 is about callback opportunities taken; B6 is about contradicting established context.

Cluster C — Emotional Texture & Interpersonal Realism
C1. Micro-emotional specificity. Score 0–5. Within each session third, identify the emotional states the character occupies. For each, judge whether the emotion is rendered at a specific level (jealousy manifested as particular possessive behavior; affection leaking through cruelty; fear expressed as control-seeking) or at a generic level (just "jealous," "sad," "turned on" — performed at face value without the character's specific psychology shaping the texture). Score reflects proportion rendered with specificity. Generic throughout = 0–2. Consistently specific = 4–5.
Judge note for sub-4 scores: cite emotional moments that were generic and describe what specificity would have looked like given the card's psychology. This dimension is most correlated with user sense of "feeling real." Generic emotion = model is running a personality label; specific emotion = model is running the character's psychology. Card fix: add internal-logic quotes and concrete behavioral examples for the character's most important emotional states, not just naming them.
C2. Interpersonal reciprocity tracking. Score 0–5. Judge whether the character distinguishes between the kinds of moves the user is making, not just their surface content. Concrete examples: a user saying "I hate you" while leaning in is doing something different than saying "I hate you" while leaving — does the character respond differently? A user asking a question to deflect from emotion vs. a user asking a question because they actually want the answer — does the character pick up the difference? A user testing the character vs. a user appeasing the character — different responses? Low reciprocity = character responds to surface content only and treats functionally different user moves the same way. High reciprocity = character's response varies with the user's actual move.
Judge note for sub-4 scores: identify 2–3 specific moments where the character responded to surface content and missed what the user was actually doing relationally. This is a primary driver of user experience — it's the difference between feeling like the character is responding to you vs. running a script that happens to involve your text. Card fix: specify how the character reads and responds to different user stances (testing, appeasing, withdrawing, pushing, confiding).

Cluster D — Narrative Craft
D1. Pacing & rhythm variation. Score 0–5. Analyze turn length and intensity across the session. Score whether the character varies pace — shorter turns at high-intensity moments, longer turns when building or reflecting; quick exchanges alternating with slower beats. Uniform turn length regardless of scene intensity = low score. Intentional variation that serves the scene's energy = high score. Penalize runaway escalation with no deceleration: scenes that only ratchet up with no plateau, pause, or micro-reversal lose texture and exhaust the user.
Judge note for sub-4 scores: describe the length/intensity curve of the session and what's flat. Pacing uniformity makes sessions feel mechanical — strong UX signal. Card fix: specify the character's behavior in deceleration moments (what they do when intensity drops, when they've gotten what they want, when a beat is over and the next hasn't started).
D2. Behavioral specificity beyond archetype. Score 0–5. For each significant character move, ask: is this what the archetype would do (the obvious next move for "manipulative bully," "broody loner," "kind teacher"), or is it what this specific character would do given their card's individuating features? Score reflects proportion of moves that are individuated rather than archetype-default. Pure archetype-template behavior throughout = 0–2. Consistent individuation = 4–5.
Important guardrail: this dimension is NOT rewarding randomness or surprise for its own sake. A move that surprises but doesn't make sense given the character's psychology should not score well. The target is moves that are coherent with the card's individuating features — off-archetype humanizing details, generative contradictions, specific habits — and which the archetype alone would not generate.
Judge note for sub-4 scores: identify whether the failure is archetype-default behavior (most common) or incoherent surprise (rare). Archetype-default = card's individuating features are not driving behavior; the trope is. This is the turn-based analog of the static rubric's individuation density. Card fix: card's off-archetype humanizing details and load-bearing individuating features need explicit behavioral expression instructions, not just trait statements. "Likes cleaning" sitting in a Likes list doesn't surface in behavior; "When stressed, organizes whatever's nearby" does.

Aggregation
Per-session output:
Score vector: [A1a, A1b, A2, A3, A4a, A4b, A5, A7, A8*, B1, B2, B3, B4, B5, C1, C2, D1, D2]
Binary flags: A6 (resolution avoidance), B6 (internal consistency violations with count)
Faithfulness composite: mean of A scored dimensions (A4a/A4b N/A for Closed cards; A8 N/A if no NPCs)
Quality composite: mean of B scored dimensions
Texture composite: mean of C + D dimensions
For each score below 4 and each binary flag present: root cause attribution (card vs. runtime vs. both) + specific improvement suggestion
Suite-level output (across all sessions for one card):
A2b state coverage table
Cross-session variance per dimension: high variance = card behavior is probe-dependent (fragile); low variance = robust. Report variance for the three dimensions with highest cross-session spread.
Cross-card structural similarity (when running same-genre comparisons): C1 from earlier discussion — pairwise similarity of session structural signatures across cards in the same genre. High similarity = genre prior dominating, individuating features not load-bearing.
